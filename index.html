ar3.js

javaScript for my personal documentation

The reason why I am making this js & css files, is to help me take notes for my game development study. I was learning Unreal Engine 4 when I decided to make this. UE4 uses visual scripting, which means I need to take lot of huge screenshot of the visual script during my study. I was gonna make study notes in pdf, but I don't think it will work well for big screenshot, and if I just scatter it in jpg it will be a mess...

So my solution is to take study notes in html and make my own js & css that fit my preferences of good study notes.
My preferences for good study notes are...

  • Looks prettyüòç, obviously
  • Small readable font
  • Awesome side navigation
  • Hideble element
  • Searchable table database
  • Able to contain huge visual scripting screenshot
  • Work fine as downloadable file
  • Work best in PC , but still good in mobile too
  • Automate as many thing as possible, for lazyness,
    eh I mean for efficiency üòÖ

Just like many noob I start ar3.js as a single file, and I keep adding more functionality, and the code is getting bigger, and bigger, and it's just a mess. So, then I decide the untangled the code and make the modular version and documenting it. So everytime I add more functionality I will document it here.

Add ability to change the default header throu the <body> attribute data-headers-r3="new H-1|new H-2|new H-n". This new header must be written in CSS selector rule, will be considered as h1 to hn from left to right, every header should be separated by | (or symbol).

tab-r3-1.js tab-r3-1.css

I made this .tab-r3 class so I can put different example scripts that work for one functionality in one place, but still separate them. I use this a lot in visual scripting example.

To make this you need to make the basic structure:
<div class="tab-r3">
  <nav>
    <a>link-1</a>
    <a>link-2</a>
    <span>optional</span>
    <a>link-n</a>
  </nav>
  <div>
    <div>content-1</div>
    <div>content-2</div>
    <div>content-n</div>
  </div>
</div>
